First Connector Setup

Use When

The AIOS needs access to the client's main source of truth.

Good Fits

  • Google Workspace or Microsoft 365
  • HubSpot or CRM
  • Stripe, QuickBooks, or accounting
  • Drive, SharePoint, Dropbox
  • Scheduling, meetings, tasks

Delivery

  • Confirm access owner and permission level.
  • Choose connector method.
  • Store secrets outside chat.
  • Test one real action.
  • Update connections.md.

First Workflow Build

Scope Rule

One workflow. One owner. One clear before-and-after.

Examples

  • Daily report generation
  • Lead cleanup or scoring
  • Missed follow-up reminders
  • Meeting summary to task list
  • Proposal draft

Decision Rule

Use deterministic automation when possible. Use AI when judgment, writing, summarizing, or classification is needed.

Knowledge Base Setup

Use When

  • Documents and SOPs are scattered.
  • Employees ask the same questions.
  • Knowledge lives in the owner's head.
  • The AIOS needs source material.

Delivery

  • Gather raw source files.
  • Convert or summarize into markdown.
  • Create indexes and relationship notes.
  • Teach the AIOS how to search it.

Dashboard Proof Of Concept

Use When

The client wants visibility across tools, revenue, operations, follow-ups, or tasks.

Approach

  • Start with a lightweight artifact or report.
  • Pull from one or two sources.
  • See whether the client actually uses it.
  • Only recommend a custom dashboard after the concept proves useful.
"Before we spend money building a full dashboard, let's prove what information is actually useful."

Team Operator Expansion

Use When

The first operator is using the AIOS and another employee has a clear use case.

Delivery

  • Decide whether to use a shared private repo.
  • Clean secrets before sharing.
  • Add operator context files.
  • Clone the AIOS on the new operator's computer.
  • Train the operator on their first use case.

Retainer / AI Operator Support

Use When

The client wants ongoing help and has more ideas than internal AI capacity.

Possible Work

  • Weekly AIOS audit
  • Monthly workflow improvement
  • Connector maintenance
  • Prompt and skill updates
  • Staff training
  • Dashboard or report maintenance

Client Success Rhythm

Daily

  • Ask the AIOS what to focus on.
  • Use it for real work, not only tests.
  • Note anything it could not answer.
  • Save repeated prompts or corrections.

Weekly

  • Run $audit.
  • Run $level-up.
  • Pick one improvement.
  • Decide whether it is context, connection, capability, or cadence.
